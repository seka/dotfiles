以下の手順でセキュリティ監査を実施してください。

## 対象の決定

- 引数なし: カレントディレクトリ全体を対象
- 引数あり: 指定パス以下を対象

## チェック観点（OWASP Top 10）

1. **インジェクション** — SQL/コマンド/LDAP インジェクション、eval の使用
2. **認証不備** — ハードコードされた認証情報、弱いセッション管理
3. **機密データ露出** — API キー・パスワードのログ出力・ハードコード
4. **アクセス制御の欠如** — 認可チェック漏れ、パストラバーサル
5. **セキュリティ設定ミス** — デフォルト設定、不要な権限、デバッグモード
6. **XSS** — 未エスケープの出力、innerHTML への直接代入
7. **安全でない依存関係** — 古いライブラリ、既知の脆弱性
8. **ロギング不足** — セキュリティイベントの記録漏れ

## 出力フォーマット

```
## セキュリティ監査レポート

| severity | カテゴリ | ファイル | 問題の概要 |
|----------|---------|---------|-----------|
| Critical | インジェクション | path/to/file.ts:42 | ユーザー入力を直接 SQL に結合 |
| High     | 機密データ露出 | config.js:10 | API キーがハードコード |
...

### 詳細

#### [ファイル:行番号]
**問題**: ...
**影響**: ...
**推奨対応**: ...
```

## severity 基準
- **Critical**: 即座に悪用可能、データ漏洩・システム侵害に直結
- **High**: 条件次第で悪用可能、重大な影響あり
- **Medium**: 攻撃の足がかりになる可能性
- **Low**: ベストプラクティス違反、直接的リスクは低い
